手把手带你搭建自己的博客

最近公司部门掀起了一股博客热,大家都在输出自己的文档,既然如此,搭建博客就作为我的第一批博客输出吧!

准备工作

安装git

不会的自行百度!

安装node.js

  • 点我下载

LTS为长期支持版，Current为当前最新版(我们选长期支持版就好啦)

  • 安装好以后, 命令行敲node -v 检验是否安装成功

下载hexo

  • 使用npm install -g hexo-cli下载时, 可能会报错, 如果报错可使用sudo npm install – unsafe-perm –verbose -g hexo
  • 安装完成后, hexo -v查看版本,检查是否安装成功
  • 找到一个你钟意的文件夹~ 命令行输入 hero init 来初始化项目!

每个文件夹的作用

  • node_models (依赖的模块包)
  • public：存放的是生成的页面
  • scaffolds：命令生成文章等的模板
  • source：用命令创建的各种文章
  • themes：主题
  • _config.yml：整个博客的配置
  • db.json：source解析所得到的
  • package.json：项目所需模块项目的配置信息

此时此刻, 你的博客雏形就已经搭建完毕了, 你可以通过启动本地来看到

本地访问地址: http://localhost:4000/

你以为这样就结束了吗! 还米有!

部署到github

  • 首先你要拥有一个github的账号, 我猜你一定有!
  • 创建一个project, 名字以你的github名字+.github.io 例如: weiyutinger.github.io
  • 创建ssh: 在命令行中输入ssh-keygen -t rsa -C "youremail@example.com,替换邮箱为你的github邮箱,一路回车
  • 在C://User//yourname//.ssh中会发现几个文件,其中yourname为你的电脑用户名
  • 打开id_rsa.pub, 复制全部内容
  • 打开github, 将复制的内容贴到key中,新增自己的ssh
  • 命令行输入ssh -T git@github.com, 输入yes, 验证是否成功
  • 打开你的blog项目, 修改_config.yml文件配置

安装一波git部署的工具npm install hero-deployer-git –save

输入命令

浏览器输入http://yourgithubname.github.io 就可以看到你的博客了

怎么样, 是不是感觉奇丑无比, 至于如何修改主题, 我们下期再见!
